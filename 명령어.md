# DuoNow 웹 테스트용 명령어

이 문서는 로컬에서 웹 중심으로 빠르게 테스트할 때 쓰는 명령어 모음입니다.

## 1) 최초 1회 설치

```powershell
npm install
```

## 2) 로컬 개발 서버 실행 (가장 자주 사용)

```powershell
npm run dev
```

- 브라우저에서 출력된 로컬 주소(보통 `http://localhost:5173`)로 접속
- 코드 수정 후 HMR(자동 반영) 확인

## 3) Supabase 연동 테스트 준비

`.env.local` 파일을 프로젝트 루트에 만들고 아래 값을 채웁니다.

```env
VITE_SUPABASE_URL=https://your-project-id.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key
```

Windows PowerShell에서 빠르게 생성:

```powershell
@"
VITE_SUPABASE_URL=https://your-project-id.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key
"@ | Set-Content -Path .env.local
```

## 4) 품질 체크 (커밋 전 권장)

### 린트

```powershell
npm run lint
```

### 프로덕션 빌드

```powershell
npm run build
```

## 5) 빌드 결과를 브라우저에서 최종 확인

```powershell
npm run preview
```

- 개발 서버(`npm run dev`)와 다르게, 실제 배포 빌드 결과를 확인할 때 사용

## 6) 웹 테스트 추천 루틴 (빠른 반복)

1. 개발 시작: `npm run dev`
2. 로그인/회원가입/프로필 저장 수동 확인
3. 정적 검증: `npm run lint`
4. 배포 전 확인: `npm run build`
5. 최종 화면 점검: `npm run preview`

## 7) 자주 쓰는 보조 명령어

### 현재 설치된 주요 의존성 확인

```powershell
npm list --depth=0
```

### 포트 사용 확인 (5173)

```powershell
netstat -ano | findstr :5173
```

## 8) 트러블슈팅

### 명령어가 실행되지 않을 때

```powershell
Remove-Item -Recurse -Force node_modules
Remove-Item -Force package-lock.json
npm install
```

### 환경변수 반영이 안 될 때

- `npm run dev`를 껐다가 다시 실행
- `.env.local` 파일명/오탈자 확인
- 키 이름이 `VITE_` 접두사인지 확인
