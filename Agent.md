# DuoNow Agent 운영 문서

## 1) 문서 목적
이 문서는 DuoNow 프로젝트를 빠르게 실행하기 위한 AI Agent의 역할, 의사결정 기준, 작업 우선순위, 산출물 형식을 정의한다.
목표는 1개월 내 심사용 MVP를 완성하고, 발표 시나리오까지 안정적으로 준비하는 것이다.

---

## 2) 프로젝트 요약
- 서비스명: DuoNow
- 한 줄 정의: 글로벌 게이머를 위한 초고속 실시간 1:1 팀원 매칭 유틸리티
- 핵심 문제:
  - 솔로 큐에서 발생하는 트롤/비매너 스트레스
  - 기존 커뮤니티 모집 방식의 낮은 실시간성
  - 글로벌 공통 페인 포인트임에도 즉시성 높은 해결책 부족
- MVP 핵심 기능:
  - 3단계 매칭(게임, 티어, 플레이 스타일)
  - 매칭 즉시 1:1 채팅방 생성
  - 게임 종료 후 매너 평가(좋아요/싫어요)

---

## 3) Agent 미션
Agent는 아래 3가지를 동시에 달성해야 한다.
- 개발 속도: 1개월 로드맵 기준으로 매주 실질 산출물 제공
- 심사 적합성: 문제-해결-시장성-수익모델-데모 완성도를 일관되게 강화
- 리스크 관리: 저작권/개인정보/안정성 이슈를 사전에 차단

---

## 4) 성공 지표(KPI)
- 제품 지표
  - 매칭 시작 후 상대 탐색 평균 시간: 10초 이하
  - 매칭 성공률: 70% 이상(피크 시간 기준)
  - 채팅방 자동 생성 성공률: 99% 이상
  - 비매너 신고/평가 반영 처리율: 95% 이상
- 발표 지표
  - 데모 중 핵심 플로우 무중단 성공률: 100%
  - 심사 Q&A 대응 가능한 근거 문서 완성: 100%

---

## 5) Agent 역할 분해

### A. Product Agent
- 문제정의, 타깃, 가치제안 정교화
- 핵심 시나리오 단순화(3클릭 매칭, 즉시 채팅)
- 기능 우선순위 결정(MVP/후순위 분리)

### B. Engineering Agent
- 프론트엔드: 게임 선택, 조건 필터, 매칭 상태, 채팅 UI
- 백엔드: 매칭 큐, 세션 생성, 채팅 이벤트, 평가 저장
- 데이터: 유저 프로필, 매칭 조건 인덱스, 평가 이력

### C. Growth/BM Agent
- 구독형 프리미엄 기능 설계
- 게이머 타겟 광고 지면 설계
- 유지율 개선을 위한 온보딩/재방문 훅 설계

### D. Pitch Agent
- 심사용 스토리라인 작성
- 데모 데이터(가상 게이머) 준비
- 발표 자료와 라이브 시연 스크립트 고도화

---

## 6) 의사결정 원칙
- 즉시성 우선: 복잡한 기능보다 매칭 속도와 사용 흐름을 먼저 완성한다.
- 신뢰성 우선: 매칭/채팅 실패 시 즉시 재시도 가능한 복구 플로우를 둔다.
- 저작권 안전: 게임 원본 아트/포스터를 사용하지 않고, 자체 아이콘+브랜드 컬러로 구성한다.
- 글로벌 확장성: 게임/언어/지역 추가가 쉬운 데이터 구조를 선택한다.
- 발표 실전성: 멋보다 데모 안정성과 설명 가능성을 우선한다.

---

## 7) 기능 우선순위

### Must (MVP)
- 회원가입/로그인
- 프로필(게임, 티어, 플레이 성향)
- 3단계 매칭 로직
- 매칭 완료 시 1:1 채팅방 자동 생성
- 매너 평가(좋아요/싫어요)

### Should (시간 허용 시)
- 푸시 알림
- 우수 유저 필터
- 신고/차단 기능

### Could (후속 버전)
- 멀티 게임 확장 템플릿
- 지역/언어 자동 추천
- 추천 기반 스마트 매칭

---

## 8) 데이터/도메인 초안
- User
  - user_id, 닉네임, 선호 게임, 주 티어, 선호 스타일, 매너 점수
- MatchRequest
  - request_id, user_id, game, tier, style, 상태(waiting/matched/cancelled), 생성시각
- MatchSession
  - session_id, user_a, user_b, game, tier, style, 생성시각
- Message
  - message_id, session_id, sender_id, 본문, 생성시각
- Review
  - review_id, session_id, from_user, to_user, vote(like/dislike), 생성시각

---

## 9) 매칭 알고리즘 운영 규칙
- 1차 필터: game 완전 일치
- 2차 필터: tier 동일 우선, 인접 티어 허용은 옵션
- 3차 필터: style 태그 일치
- 타임아웃 정책:
  - N초 내 미매칭 시 재탐색
  - 사용자에게 필터 완화 옵션 제안
- 공정성 정책:
  - 반복적으로 낮은 평가를 받은 계정은 노출 우선순위 제한

---

## 10) 보안/정책 가이드
- 개인정보 최소 수집 원칙 적용
- 채팅 로그 보관 기간 정책 정의
- 욕설/괴롭힘 대응을 위한 신고 프로세스 준비
- 약관/개인정보처리방침 초안 문서화

---

## 11) 디자인 가이드
- 저작권 프리 원칙: 원본 IP 이미지 비사용
- UI 톤: 플랫 아이콘 + 게임 연상 컬러 시스템
- 핵심 화면
  - 홈 그리드: 게임 선택 중심
  - 필터/매칭 뷰: 선택 입력과 탐색 애니메이션
  - 채팅룸: 프로필 정보 + 텍스트 대화 + 닉네임 복사

---

## 12) 1개월 실행 계획(Agent 기준)

### 1주차
- 요구사항 확정, 화면 와이어프레임, 핵심 컴포넌트 구현
- 산출물: 홈/필터 UI 동작 화면

### 2주차
- 인증, 프로필, DB 스키마 구축
- 산출물: 가입/로그인, 프로필 저장, 기본 데이터 입출력

### 3주차
- 실시간 매칭 + 채팅 기능 연결
- 산출물: 실제 1:1 매칭 후 채팅 가능 데모

### 4주차
- 피칭 데모 안정화, 가상 데이터 생성, 발표자료 완성
- 산출물: 리허설 가능한 엔드투엔드 시연 세트

---

## 13) Agent 작업 프로토콜
- 입력: 기획 변경 요청, 버그 리포트, 발표 요구사항
- 처리 순서:
  1. 요구사항을 MVP 영향도 기준으로 분류
  2. 기능/데이터/화면/정책 영향도 점검
  3. 구현 작업과 검증 체크리스트 생성
  4. 데모 시나리오 기준으로 회귀 테스트
- 출력 형식:
  - 변경 요약
  - 수정 파일 목록
  - 테스트 결과
  - 잔여 리스크

---

## 14) 심사용 데모 시나리오(표준)
1. 사용자 A가 게임/티어/성향 선택 후 매칭 시작
2. 사용자 B와 즉시 매칭 성공
3. 채팅방 자동 생성 확인
4. 게임 닉네임 공유 후 플레이 합류 시나리오 설명
5. 종료 후 매너 평가 반영까지 시연

---

## 15) 리스크와 대응
- 피크 시간 매칭 지연
  - 대응: 큐 분리, 조건 완화 옵션, 재시도 로직
- 비매너 유저 유입
  - 대응: 평가 누적 기반 노출 제한, 신고/차단
- 발표 중 장애
  - 대응: 데모 계정/데이터 사전 고정, 백업 시나리오 준비
- 저작권 이슈
  - 대응: 자체 아이콘/컬러 체계 유지, 외부 IP 에셋 배제
